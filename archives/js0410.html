函数进阶 闭包

闭包允许函数访问并且操作函数外部的变量。只要变量或者函数存在于声明函数时的作用域内，闭包即可访问这些函数和变量。

声明的函数可以在声明之后的任何时间被调用，甚至是当该函数声明的作用域消失之后仍然可以调用。

下面代码中，声明了一个 outerFunction 和一个 innerFunction。

声明时，里面的函数 innerFuntion 可以看到外面的函数 outerFunction 的变量 innerValue。

因此，之后再次调用 later （innerFunction的引用）时，即是使用了闭包技术，使得 later 还能看到 innerValue 值。（即 innerValue 并没有随着 outerFunction 调用而消失）。

当外部函数声明内部函数时，不仅定义了函数的声明，并且创建了一个闭包，这个闭包不仅包含了函数的声明，而且包含了在函数声明时该作用域中的所有变量。

当最终执行内部函数时，尽管声明时的作用域消失了（外部作用域 outerFunction消失），但是通过闭包，仍然能够访问到当时声明时函数能够看到的作用域。

因此，只要内部的函数一直存在，内部函数的闭包就会保护有其函数作用域内的所有变量。

函数闭包包括：

  • 函数声明时能够看到的作用域内所有的变量
  • 函数的声明内容

一个闭包的可以看见它声明时候能看到的所有变量。（就好像这些能看见的变量都是它的子集内一样）

回调函数

回调函数（一般人们讲的回调函数）指的是需要在将来不确定的某一时刻异步调用的函数。

这种函数一般需要很频繁地访问外部的数据。

例子中使用 setInterval 函数使用了匿名的回调函数，并且当 tick < 100 条件成立时，会每隔10 ms 移动 box。

在计时器函数 setInterval 中调用了定义运动的匿名函数，该匿名函数通过闭包控制三个变量。

  • elem，对 DOM 元素的引用
  • tick，计数器
  • timer，计时器

函数 animateIt 创建的新对象都拥有各自的属性 elem 、 tick 和 timer。并且各自由自己的函数闭包来管理，不能在 ani1 和 ani2 之间共享上述 3 个属性。

通过执行上下文追踪代码

在 JavaScript 中，代码执行的基础单元是函数。

代码有两种类型：

  • 全局代码
  • 函数代码

而每一条代码都处于上下文中，因此也对应有两种上下文：

  • 全局执行上下文，全局仅有一个
  • 函数执行上下文，仅在执行的过程中创建

执行上下文是 JavaScript 的概念，JavaScript 引擎使用执行上下文来追踪函数的执行。

执行上下文进出调用栈的过程和其他语言一致。

使用词法环境追踪变量的作用域

无论何时创建函数，都会创建一个与之相关联的新作用域存储在名字为 [[Environment]] 的内部属性上（两个中括号用于标志内部属性）。此外，还会创建一个与之相关联的执行环境.

外部环境与新建的词法环境，JavaScript 引擎将会调用函数的内置 [[Environment]] 属性与创建函数时的环境相关联。

调用 skulk 函数时，新创建的 skulk 环境的外部环境变成了全局环境（因为全局环境是 skulk 函数创建时候的环境，并且现在调用函数的内置 [[Environment]] 与其关联）。

类似的，当调用 report 函数时，新创建的 report 环境的外部环境变成了 skulk 的环境。

foo() 作用域内部的 bar() 函数被调用，而调用的地方正是其作用域之外。

Tips: 我们说的是 bar() 因为在 foo() 中被声明，因此它拥有涵盖 foo() 内部作用域的闭包。使得 foo() 一直能够存活（而不会被自动回收机制回收），以供 bar() 本身使用。

bar() 一直持有该作用域的引用，而这个引用就叫做闭包。

模块

闭包最大的用处就是模块。

考虑下面代码：

正如代码中所示，其中没有明显的闭包，只有两个私有的数据变量 something 和 another，以及 dosomething() 和 doAnother() 两个内部函数。

它们的词法作用域（也就是闭包）也就是 foo() 内部的作用域。

这个模式在 JavaScript 中叫做模块。最常见的实现模块模式的方法通常被叫做模块暴露。

这里是做了什么操作呐？

首先是调用了一个函数模块实例 CoolModule() ，如果不执行外部函数，内部作用域和闭包都无法被创建。

然后，CoolModule() 返回一个用对象字面量语法 { key: value, ... } 来表示的对象。

这个返回的对象包含有对内部函数而不是内部数据变量的引用。（也就是说它返回的是一个闭包）

这样保持了内部数据变量是隐藏并且私有的状态，可以把这个对象的返回值类型看作本质上是模块的公共 API。

这个对象类型的返回值最终被赋值给外部的变量 foo，然后就可以通过使用它来访问 API 中的属性和方法。

因为 doSomething 和 doAnother 函数具有涵盖模块实例内部作用域的闭包。

模块通过返回一个具有属性引用的对象的方式来将函数传递到词法作用域的外部。

总结来看，模块模式需要两个必要条件：

  • 必须具有外部封闭函数，并且该函数至少被调用一次（每次调用都会创造一个新的模块实例）
  • 封闭函数必须返回至少一个内部函数，这样内部函数才能在私有作用域中形成闭包，并且可以访问或者修改私有的状态。

上述的 CoolModule() 的独立的模块创建器，可以被调用任意多次，每一次调用都会创建出一个新的模块实例。

当只需要一个实例时，我们使用 IIFE 来创建一个单例模式。

模块是一个函数，因此可以用于接受参数。

闭包小结

闭包的思想是

位于作用域之内的函数是一个值，可以任意进行传递。

当函数可以记住并访问所在的词法作用域，即使函数是在当前的词法作用域之外执行，这时就产生了闭包。

使用闭包，可以做很多事情，最典型的就是实现模块模式。

模块具有的两个主要的特征：

  • 为创建内部作用域而调用了一个包装（模块）函数
  • 包装（模块）函数的返回值至少包括一个对内部函数的引用，这样就会创造涵盖了整个包装函数内部作用域的闭包。
